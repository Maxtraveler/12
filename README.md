# Система проверки промокодов «Пятёрочка» с уведомлением в Telegram

## Требования

- Python 3.11+
- Установленный `pip`

## Установка

1. Создайте и активируйте виртуальное окружение (опционально).
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` на основе `.env.example` и заполните:

- `BOT_TOKEN` — токен вашего Telegram-бота.
- `OWNER_ID` — ваш Telegram ID.
- `TARGET_URL` — URL проверки промокода (см. ниже раздел про DevTools).
- `REQUEST_DELAY_MIN`, `REQUEST_DELAY_MAX` — задержки между запросами.

## Как найти реальный запрос проверки промокода через DevTools

1. Откройте сайт или веб-версию «Пятёрочки» в браузере (Chrome/Firefox).
2. Нажмите `F12`, перейдите во вкладку **Network (Сеть)**.
3. Убедитесь, что включён фильтр **XHR / Fetch** (зависит от браузера).
4. Введите любой промокод в поле на сайте и нажмите кнопку проверки/активации.
5. В списке запросов найдите тот, который отвечает за проверку промокода:
   - Обычно это запрос к API (URL типа `/promo` / `/promocode` / `/check`).
6. Кликните по запросу и:
   - На вкладке **Headers (Заголовки)**:
     - Скопируйте полный **Request URL** — вставьте его в `TARGET_URL` в `.env`.
     - Обратите внимание на заголовки `User-Agent`, `Content-Type`, `Accept`, `Origin`, `Referer`, `Cookie` и возможные токены авторизации.
   - На вкладке **Request Payload** / **Form Data**:
     - Посмотрите, в каком поле передаётся промокод (например, `promocode`, `code` и т.п.).
     - При необходимости адаптируйте словарь `payload` в `core/checker_engine.py` под эту структуру.
   - На вкладке **Response (Ответ)**:
     - Найдите признаки успешного кода (поле `success`, `status`, `discount` и т.д.).
     - При необходимости подкорректируйте логику разбора ответа в `core/checker_engine.py`.

> Важно: токены авторизации и cookies могут истекать, следите за актуальностью данных.

## Запуск бота

1. Убедитесь, что в `.env` корректно заполнены все поля.
2. Запустите:

```bash
python main.py
```

3. Откройте диалог с вашим ботом в Telegram и отправьте:
   - `/start` — приветствие.
   - `/start_check` — запуск выбора режима проверки:
     - **Маска** — генерация кодов по маске (например, `KL6XXXXX`).
     - **Файл** — загрузка файла с кодами (по одному коду в строке).
     - **Одиночный код** — проверка одного конкретного кода.
   - `/stats` — просмотр текущей статистики.

## Результаты

- Рабочие промокоды (HIT) сохраняются в файл `hits.txt` с указанием времени и размера скидки (если удалось определить).
- Текущая статистика (проверено, найдено, процент успеха) доступна по команде `/stats` и может приходить периодически во время работы.
